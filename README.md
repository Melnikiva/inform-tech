# Інформаційно-пошукові системи та сервіси

### 1. Моделі і методи зберігання даних

Для збереження даних використовують різні типи та моделі баз даних. Можемо розділити існуючі моделі збереження даних на такі групи:

- **Прості типи баз даних**  
  Бази даних із простою структурою.

  - _Текстові файли_  
    .txt, .csv фали тощо
  - _Ієрархічні бази даних_  
    Файлова система ОС, деревоподібна структура
  - _Мережеві бази даних_  
    Розширює ієрархічну систему, структура має вигляд графу, а не дерева

- **Реляційні бази даних**  
   Дані формуються у таблиці з **рядків та стовпців**. У рядках наводяться відомості про об’єкти, а стовпці є властивостями об’єктів.  
   Відносини об’єктів у реляційних БД моделюються за допомогою **зовнішніх ключів** — посилань на інші таблиці.  
  Запити в реляційних базах даних формують за допомогою структурованої **мови SQL**.  
  (MySQL, PostgreSQL тощо)
- **Нереляційні бази даних**  
   Призначені для зберігання неструктурованих даних, таких як електронні листи, відео, зображення та документи, які важко стандартизувати.
  - _Документоорієнтовані БД_  
    Дані зберігаються в структурованих форматах — XML, JSON, BSON. При цьому зберігається адресний доступ до даних за ключем. (MongoDB, DocumentDB)
  - _Бази даних «Ключ-значення»_  
    Дані зберігаються як словник, де покажчиком є ключ. (DynamoDB, Redis)
  - _Графові бази даних_  
    Зв’язками виступають ребра графа, а самі об’єкти — це вузли чи вершини. (Neo4J)
  - _Стовпчикові, або колонкові бази даних_  
    Замість таблиць тут використовуються колонкові сімейства. Технологію активно використовують при побудові аналітичних систем та сервісів, що працюють з великими обсягами даних.
  - _Бази даних часових рядів_  
    Такі БД можна використовувати для відстеження історичної динаміки за низкою показників. Тут дані групуються за часовими мітками. (TimescaleDB)
  - _Бази даних NewSQL_  
    Особливостями цього типу БД є широкі можливості масштабування та велика продуктивність і доступність даних. Однак є високі вимоги до апаратних ресурсів розробників. (MemSQL, VoltDB)
  - _Багатомодельні бази_  
    Такі БД поєднують у собі кілька підходів до організації даних одночасно. (ArangoDB)

### 2. Класифікація інформаційних систем і місце серед них інформаційно-пошукових систем

**Інформаційні системи** можна класифікувати за різними ознаками, такими як режимом пошуку, масштабністю, типом інформаційно-пошукової мови, типом критерію відповідності, ступенем автоматизації та іншими.

**Інформаційно-пошукові системи** є однією з автоматизованих інформаційних систем, а яких не передбачається завершальна обробка даних. Ці системи призначені для пошуку текстів (документів, їх частин, фактографічних записів) в сховищах даних (БД) за формальними характеристиками.

### 3. Організація пошуку. Пошукові машини.

**Організація пошуку** включає в себе розробку алгоритмів пошуку, які використовуються пошуковими машинами.

**Пошукові машини** - програмно-апаратні комплекси, які забезпечують функціональність пошукової системи, здійснюють автоматизацію пошуку в комп’ютерній мережі та виконують оперативне оновлення інформації у своїй базі даних без участі людей.

### 4. Створення і типи індексів

**Індекс** - це об’єкт бази даних, що створений з метою підвищення ефективності виконання запитів

Типи індексів:

- _B-tree індекс_  
  Застосовується для реляційних баз даних.
  Забезпечує швидкий пошук і вставку.
- _Хеш-індекс_  
  Використовує хеш-функцію для зберігання ключів та значень.
  Ефективний для рівномірно розподілених даних.
- _Bitmap-індекс_  
  Зберігає bitmap для кожного можливого значення поля.
  Використовується для невеликої кількості унікальних значень.
- _GiST (Generalized Search Tree)_  
  Використовується для геопросторових та інших складних типів даних.

Наприклад, у SQL можна створити індекси наступним чином:

- Одиничний індекс

```sql
CREATE INDEX index_name ON table_name (column_name)
```

або

- Складений індекс

```sql
CREATE INDEX index_name ON table_name (column1, column2);
```

### 5. Проблеми індексування

Можемо виділити такі проблеми індексування:

- _Необхідна додаткова памʼять_
- _Затримки при операціях вставки, оновлення та видалення_  
  Під час цих операція індекси також повинні бути оновлені
- _Вибір правильного типу індексу_  
  Вибір невідповідного індексу може зменшити ефективність індексування
- _Необхідність регулярної оптимізації для підтримки високої продуктивності_

### 6. Запити до пошукових машин

**Запити до пошукових машин** - фрази або ключові слова, які користувач вводить у пошуковий двигун для отримання відповіді або списку відповідних результатів. Запити можуть бути короткими ключовими словами чи довшими фразами, з яких вже будуть виділятись ключові слова.

### 7. Якість роботи пошукачів

Якість пошукових систем можна оцінити за деякими критеріями:

- _Точність результатів_
- _Швидкість пошуку_
- _Фільтрація результатів_
- _Адаптація до різних мов_
- _Наявність великої кількості документів для пошуку_
- _Також у деяких інформаційних системах можливий пошук за мультимедійними даними_

### 8. Посилальне ранжування (Page Rank)

**Page Rank** - алгоритм, який використовується пошуковими машинами для визначення релевантності веб-сторінок на основі їх взаємозв’язків. Алгоритм був розроблений компанією Google і з тих пір став одним з найбільш важливих критеріїв ранжування веб-сторінок.

Алгоритм бере за основу те, що інтернет представляється у вигляді графа, де вузли — це веб-сторінки, а ребра — гіперпосилання між сторінками.

PageRank працює на основі того, що посилання на сторінку можуть розглядатися як голоси за неї. Чим більше посилань вказує на сторінку, тим вище її PageRank, однак, не всі посилання мають однакову вагу. PageRank враховує якість посилань, що вказують на сторінку, а також їхню кількість. Інші фактори, такі як вік сторінки, її релевантність до запиту користувача, а також її вміст, також впливають на рейтинг сторінки.

Загалом, алгоритм PageRank допомагає пошуковим машинам визначати, які сторінки є найбільш релевантними для конкретного запиту користувача.

### 9. Поняття інформації як категорії, дані і знання

**Дані** - набір певних фактів чи спостережень, які є подальшим матеріалом для створення інформації. Дані можуть бути числовими, текстовими, графічними тощо. Вони представляють собою "сирі" факти, які потребують обробки та інтерпретації, щоб стати корисними.

**Інформація** - дані, які були оброблені та відсортовані для отримання певного сенсу або значення. Інформація виникає, коли дані організовані, інтерпретовані та контекстуалізовані. Інформація має сенс і може бути використана для прийняття рішень чи розуміння певного явища.

**Знання** - більш високий рівень розуміння та використання інформації. Знання включають в себе вміння аналізувати, синтезувати та використовувати інформацію для вирішення проблем, прийняття рішень та досягнення цілей. Знання формується на основі інформації та досвіду.

Таким чином утворюється ієрархія:
_Дані -> Інформація -> Знання_

### 10. Програмне та апаратне забезпечення для організації пошуку інформації в мережі інтернет

Розглянемо кілька аспектів програмного та апаратного забезпечення, що використовуються у процесі пошуку в інтернеті:

**Програмне забезпечення:**

- _Пошукові двигуни_  
  Пошукові двигуни, такі як Google, Bing тощо, використовують складні алгоритми для індексації та ранжування мільярдів веб-сторінок.
- _Модулі фільтрації та аналізу_  
   Модулі для фільтрації спаму, аналізу вмісту та визначення релевантності результатів.
- _Алгоритми ранжування_  
  Такі алгоритми оцінюють різні фактори, такі як PageRank, релевантність ключових слів тощо.
- _Оновлення та покращення_  
  Постійні оновлення та покращення алгоритмів пошукових систем, щоб вони краще відповідали змінюючимся потребам користувачів та враховували нові тенденції.

**Апаратне забезпечення**

- _Мережеве обладнання_  
  Маршрутизатори та комутаторимаршрутизатори та комутатори
- _Сервери та датацентри_  
  Великі пошукові двигуни мають величезні серверні ферми та датацентри, які забезпечують обробку великої кількості запитів в реальному часі.
- _Системи зберігання даних_  
  Великі об'єми індексованої інформації вимагають ефективних та великих систем зберігання даних
- _Обчислювальні потужності_
  Використання високопродуктивних серверів та комп'ютерів, оснащених потужними процесорами та оперативною пам'яттю, для ефективного обчислення та обробки запитів
